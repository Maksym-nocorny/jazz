# Anti-AI design — quick checklist

A printable summary of the `anti-ai-design` skill (Law 1). Judge against the project's **chosen
design direction**, not as a blanket ban — deliberate exceptions are allowed if logged in
`DECISIONS.md`.

## Avoid (default AI tells)
- [ ] Default purple→blue / indigo gradient as the brand look
- [ ] Generic identical three-card feature grid
- [ ] Decorative glassmorphism (frosted cards for vibe)
- [ ] Default Inter / Roboto / Poppins with no intent
- [ ] Emoji used as icons / bullets / section markers
- [ ] 3D blobs, gradient-mesh "AI art," faceless 3D figures
- [ ] One uniform border-radius applied to everything
- [ ] Low-contrast gray body text (fails WCAG AA)
- [ ] Perfectly even, identical section rhythm
- [ ] Shipped lorem ipsum / placeholder copy
- [ ] Glossy, generic AI/stock imagery

## Prefer (bespoke moves)
- [ ] Intentional type pairing chosen for the direction (tokenized)
- [ ] Editorial / asymmetric layout with real hierarchy
- [ ] A distinctive, semantic palette with a reason
- [ ] Real content from the copy deck — never placeholder
- [ ] Consistent custom/system icon set
- [ ] Purposeful motion (feedback / progress / reveal)
- [ ] Deliberate spacing, radius, and shadow scales

## Gate questions
1. Could "make me a modern landing page" have produced this? Add intent.
2. Is every color/font/gradient a deliberate, tokenized choice for the direction?
3. Real point of view in layout/type/color — or just safe and symmetrical?
4. Contrast WCAG AA? Icons non-emoji and consistent? Content real?
5. Any "avoid" item present — justified by the direction and logged?
